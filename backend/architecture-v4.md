## 1. 架构总览

### 三层架构模型

│  LAYER 1: 交互层                                                  │
│  └── 交互Agent                                                    │

│       • 结果汇总展示 → 整合各域返回结果                            │

│  LAYER 2: 规划层                                                  │
│  └── 任务规划Agent                                                │

│       • 拆分任务 → 生成各域子任务清单                              │
│       • 分发任务 → 调用各域知识Agent                               │
└──────────────────────────┬──────────────────────────────────────┘
                           ↓ 子任务分发
┌─────────────────────────────────────────────────────────────────┐
│  LAYER 3: 执行域（3个域，每个域有独立知识Agent）                    │
│                                                                 │

│  │      工程内容执行区      │  │         信息收集记录区        │   │

## 2. Agent列表

| 层级 | 域 | Agent名称 | 职责 | 置信度输出 |
|------|-----|-----------|------|-----------|
| L1 | - | 交互Agent | 接收输入、展示结果 | 无 |
| L2 | - | 任务规划Agent | 拆分任务、分发到各域 | 无 |
| L3 | 定额域 | **定额域知识Agent** | 管理定额知识、分发4步任务 | 每步评估 |
| L3 | 定额域 | 清单初步整理Agent | 提取项目特征、识别专业 | 是 |
| L3 | 定额域 | 清单标准化整理Agent | 规范名称、补充特征 | 是 |
| L3 | 定额域 | 定额套取Agent | 定额匹配、单价获取 | 是 |
| L3 | 定额域 | 复核Agent | 合理性检查、汇总输出 | 是 |
| L3 | 查询域 | **查询域知识Agent** | 管理查询知识、分发任务 | 评估 |
| L3 | 查询域 | 查询域执行Agent | 招标/价格/历史/记录查询 | 是 |
| L3 | 资料域 | **资料域知识Agent** | 管理资料知识、分发任务 | 评估 |
| L3 | 资料域 | 资料域执行Agent | 报告/文档/整理/转换 | 是 |
| 全局 | - | 跨域知识协调Agent | 汇总结果、冲突处理、人工介入 | 全局评估 |

## 3. 定额域 4步流水线

```
原始清单/图纸/描述
       ↓
┌─────────────────────────────────────────────────────────────┐
│ Step 1: 清单初步整理Agent                                     │
│ • 识别项目类型、专业、分部分项                                 │
│ • 提取原始工程量数据                                          │
│ • 标记缺失信息                                                │
│                    ↓ 输出: 初步整理清单 + 置信度               │
├─────────────────────────────────────────────────────────────┤
│ Step 2: 清单标准化整理Agent                                    │
│ • 按定额规范统一项目名称、单位、规格                           │
│ • 补充工程特征描述                                            │
│ • 处理换算、系数调整                                          │
│                    ↓ 输出: 标准化清单 + 置信度                 │
├─────────────────────────────────────────────────────────────┤
│ Step 3: 定额套取Agent                                         │
│ • AI匹配定额编码                                              │
│ • 获取定额单价                                                │
│ • 计算合价                                                   │
│                    ↓ 输出: 定额匹配结果 + 置信度               │
├─────────────────────────────────────────────────────────────┤
│ Step 4: 复核Agent                                             │
│ • 检查定额套用合理性                                          │
│ • 核对工程量计算                                              │
│ • 汇总造价                                                   │
│                    ↓ 输出: 最终成果 + 总置信度                 │
└─────────────────────────────────────────────────────────────┘
        ↑ 定额域知识Agent（全程参与，每步辅助+置信度评估）
```

## 4. 置信度与人工介入机制

| 置信度 | 处理方式 |
|--------|----------|
| ≥90% | 自动通过，记录日志 |
| 70-89% | 标注"建议复核"，继续执行，汇总时提示 |
| <70% | **【阻塞】**，必须人工介入，确认后继续 |

## 5. 工作流程示例

```
用户: "学校综合楼造价多少？"

L1-交互Agent → 转化为任务描述

L2-任务规划Agent:
  • 分析：涉及建筑+安装专业，需要定额套取和价格查询
  • 拆分：
    - 子任务A(定额域): 建筑部分清单定额套取
    - 子任务B(定额域): 安装部分清单定额套取
    - 子任务C(查询域): 查询当地人工材料价格
    - 子任务D(资料域): 汇总生成造价报告
  • 分发：同时调用定额域/查询域/资料域 知识Agent

L3-各域并行执行:
  定额域知识Agent → 分发Step1-4给域内执行Agent
  查询域知识Agent → 分发查询任务给执行Agent
  资料域知识Agent → 等待前两域结果后生成报告

跨域知识协调Agent → 汇总各域结果，返回给交互Agent
```

## 6. 与现有Backend对照

### 现有Agent（main.py中）

| 现有名称 | 对应新架构 | 状态 |
|---------|-----------|------|
| coordinator | 跨域知识协调Agent | 需重构 |
| building | 定额域知识Agent + 执行Agent | 需拆分 |
| installation | 定额域知识Agent（安装部分） | 合并到定额域 |
| municipal | 定额域知识Agent（市政部分） | 合并到定额域 |
| garden | 定额域知识Agent（园林部分） | 合并到定额域 |
| antique | 定额域知识Agent（仿古部分） | 合并到定额域 |

### 需新增Agent

- 交互Agent (L1)
- 任务规划Agent (L2)
- 定额域知识Agent (L3)
- 定额域4个Step执行Agent (L3)
- 查询域知识Agent + 执行Agent (L3)
- 资料域知识Agent + 执行Agent (L3)
