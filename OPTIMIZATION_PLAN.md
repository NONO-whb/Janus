## 1. 薄客户端架构边界

| 职责 | 前端（Flutter） | 后端（FastAPI） |
|------|----------------|----------------|
| **UI渲染** | ✅ 所有界面组件 | ❌ |
| **用户交互** | ✅ 点击、滑动、手势 | ❌ |
| **状态管理** | ✅ 页面级状态 | 全局业务状态 |
| **数据映射** | ✅ DTO ↔ UI Model | Entity ↔ DTO |
| **复杂计算** | ❌ | ✅ 定额匹配、工程量计算 |
| **AI编排** | ❌ | ✅ Agent调度、流式响应 |
| **文件存储** | 临时缓存 | ✅ 持久化存储 |
| **数据持久化** | 本地缓存（可选） | ✅ 数据库/向量库 |

## 3. 优化策略（按薄客户端原则）

### 3.1 前端优化（纯UI层面）

| 优化项 | 当前问题 | 优化方案 | 优先级 |
|--------|---------|---------|--------|
| **列表虚拟化** | 长列表卡顿 | 使用 `ListView.builder` + `AutomaticKeepAlive` | P1 |
| **图片懒加载** | 图片消息多时点慢 | `FadeInImage` + 缩略图优先 | P1 |
| **骨架屏优化** | 现有骨架过于简单 | 按内容类型定制骨架 | P2 |
| **动画性能** | 思考动画占用CPU | 使用 `RepaintBoundary` 隔离 | P2 |
| **内存管理** | 大图可能导致OOM | 图片缓存上限 + 自动清理 | P2 |

### 3.2 后端优化（薄客户端重点）

| 优化项 | 说明 | 前端配合 |
|--------|------|---------|
| **API响应压缩** | Gzip/Brotli压缩 | 自动处理 |
| **分页加载** | 消息/清单分页 | 实现无限滚动 |
| **增量更新** | WebSocket推送变更 | 局部刷新 |
| **边缘缓存** | CDN缓存静态资源 | 合理设置缓存头 |

### 3.3 映射函数优化

**薄客户端核心原则：映射函数只转换，不计算**

```dart
// ✅ 正确：纯映射，无业务逻辑
BillItemDto toDto(BillItemUiModel ui) => BillItemDto(
  code: ui.code,
  name: ui.name,
  quantity: ui.quantity, // 直接使用后端给的值
  unitPrice: ui.unitPrice,
);

// ❌ 错误：包含业务计算
BillItemDto toDto(BillItemUiModel ui) => BillItemDto(
  code: ui.code,
  total: ui.quantity * ui.unitPrice * ui.discount, // 应该后端算好
);
```

## 4. iPhone测试流程

### 4.1 开发环境测试
```bash
# 1. 检查设备连接
flutter devices

# 2. 运行到iPhone（需要数据线连接）
flutter run -d ios

# 3. 或者指定设备ID
flutter run -d "iPhone 15 Pro"
```

### 4.2 发布测试（TestFlight）
```bash
# 1. 构建Release版本
flutter build ios --release

# 2. 打开Xcode
open ios/Runner.xcworkspace

# 3. Xcode中配置签名 + Archive
# Product → Archive → Distribute App → TestFlight
```

### 4.3 Web测试（推荐用于快速验证）
```bash
# 本地网络测试（iPhone Safari访问）
flutter run -d web-server --web-hostname 0.0.0.0 --web-port 8080

# 然后在iPhone Safari访问 http://<电脑IP>:8080
```
